# دليل تشغيل مشروع SQLi-IDPS ورفعه إلى GitHub

تهانينا على تنظيم مشروعك! باتباعك للهيكل الجديد، أصبح مشروعك أكثر احترافية وسهولة في الإدارة. إليك الخطوات التفصيلية لتشغيل المشروع ورفعه.

---

## أولاً: تشغيل المشروع محلياً (Local Setup)

للتأكد من عمل المشروع بشكل صحيح، اتبع الخطوات التالية:

### 1. إعداد البيئة الافتراضية (Virtual Environment)
من المفضل دائماً استخدام بيئة معزولة لتجنب تضارب المكتبات:
```bash
# إنشاء البيئة
python -m venv venv

# تفعيل البيئة (Windows)
venv\Scripts\activate

# تفعيل البيئة (Linux/Mac)
source venv/bin/activate
```

### 2. تثبيت التبعيات
قم بتثبيت المكتبات اللازمة باستخدام ملف `requirements.txt` الذي أعددناه:
```bash
pip install -r requirements.txt
```

### 3. تشغيل التطبيق
الآن، يمكنك تشغيل المشروع من المجلد الرئيسي مباشرة:
```bash
python main.py
```
*   سيقوم التطبيق بإنشاء مجلد `instance` وقاعدة البيانات تلقائياً.
*   سيظهر لك رابط (غالباً `http://127.0.0.1:5000`)، قم بفتحه في المتصفح.
*   استخدم بيانات الدخول الافتراضية:
    *   **Username:** admin
    *   **Password:** password

### 4. تفعيل خدمة Nginx
من داخل لوحة التحكم في التطبيق، عند تفعيل الخدمة، سيقوم الكود باستدعاء `RunNginx.py` الذي سيقوم بدوره بتشغيل Nginx الموجود في مجلد `nginx`. تأكد من وجود ملف `nginx.exe` في مساره الصحيح.

---

## ثانياً: خطوات الرفع إلى GitHub بشكل احترافي

لرفع المشروع، سنستخدم أوامر `git` الأساسية. تأكد من وجود ملف `.gitignore` في المجلد الرئيسي لمنع رفع الملفات غير الضرورية.

### 1. تهيئة المستودع المحلي
افتح Terminal في المجلد الرئيسي للمشروع ونفذ:
```bash
git init
```

### 2. إضافة الملفات
قم بإضافة جميع الملفات (سيقوم `.gitignore` باستثناء الملفات غير المرغوبة تلقائياً):
```bash
git add .
```

### 3. تسجيل أول نسخة (Commit)
```bash
git commit -m "Initial commit: Professional structure for SQLi-IDPS project"
```

### 4. الربط مع مستودع GitHub
*   اذهب إلى حسابك في GitHub وأنشئ مستودعاً جديداً (New Repository) باسم `SQL-Injection-IDPS`.
*   لا تقم بإضافة README أو License (لأننا أعددناها بالفعل).
*   انسخ رابط المستودع ونفذ الأوامر التالية:
```bash
git branch -M main
git remote add origin [رابط_المستودع_الخاص_بك]
git push -u origin main
```
